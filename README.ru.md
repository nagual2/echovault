EchoVault даёт вашему агенту персистентную память. Каждое решение, исправление бага и извлечённый урок сохраняется локально и автоматически появляется в будущих сессиях. Ваш агент становится лучше чем больше вы его используете.

### Зачем я это создал

Агенты кодирования забывают всё между сессиями. Они переоткрывают те же паттерны, повторяют те же ошибки и забывают решения, принятые вчера. Я пробовал другие инструменты вроде Supermemory и Claude Mem — оба отличные, но не подошли для моего кейса.

Supermemory сохраняет всё в облаке, что было проблемой, так как я работаю с несколькими компаниями консультантом и не хочу хранить решения по кодовой базе удалённо. Claude Mem заставлял сессии потреблять слишком много памяти, что мешало запускать несколько сессий одновременно.

Я создал EchoVault чтобы решить это: локальная персистентная память для агентов кодирования, простая, быстрая и приватная.

## Возможности

**Работает с 4 агентами** — Claude Code, Cursor, Codex, OpenCode. Одна команда настраивает MCP конфиг для вашего агента.

**Нативный MCP** — Работает как MCP сервер, предоставляя `memory_save`, `memory_search` и `memory_context` как инструменты. Агенты вызывают их напрямую — без хуков shell.

**Локальный-первый** — Всё остаётся на вашей машине. Воспоминания хранятся как Markdown в `~/.memory/vault/`, читаемые в Obsidian или любом редакторе. Никакие данные не покидают вашу машину, если вы не включите облачные эмбеддинги.

**Нулевая стоимость простоя** — Нет фоновых процессов, нет демона, нет нагрузки на RAM. MCP сервер запускается только когда агент его стартует.

**Гибридный поиск** — Поиск по ключевым словам FTS5 работает из коробки. Добавьте Ollama или OpenAI для семантического векторного поиска.

**Редакция секретов** — 3-слойная редакция удаляет API ключи, пароли и credentials прежде чем что-то попадёт на диск. Поддерживает явные теги `<redacted>`, детекцию паттернов и кастомные правила `.memoryignore`.

**Кросс-агентный** — Воспоминания сохранённые Claude Code доступны для поиска в Cursor, Codex и OpenCode. Одно хранилище, много агентов.

**Совместим с Obsidian** — Файлы сессий — валидный Markdown с YAML frontmatter. Настройте Obsidian на `~/.memory/vault/` и просматривайте память агента визуально.

## Установка

```bash
pip install git+https://github.com/mraza007/echovault.git
memory init
memory setup claude-code   # или: cursor, codex, opencode
```

Готово. `memory setup` автоматически устанавливает конфиг MCP сервера.

По умолчанию конфиг устанавливается глобально. Для установки в конкретный проект:

```bash
cd ~/my-project
memory setup claude-code --project   # пишет .mcp.json в корень проекта
memory setup opencode --project      # пишет opencode.json в корень проекта
memory setup codex --project         # пишет .codex/config.toml + AGENTS.md
```

### Настройка эмбеддингов (опционально)

Эмбеддинги включают семантический поиск. Без них вы всё равно получаете быстрый поиск по ключевым словам через FTS5.

Сгенерируйте стартовый конфиг:

```bash
memory config init
```

Создаёт `~/.memory/config.yaml` с разумными умолчаниями:

```yaml
embedding:
  provider: ollama              # ollama | openai
  model: nomic-embed-text
  # base_url: http://localhost:11434   # для ollama; для openai: https://api.openai.com/v1
  # api_key: sk-...                    # требуется для openai

enrichment:
  provider: none                # none | ollama | openai

context:
  semantic: auto                # auto | always | never
  topup_recent: true
```

**Что делает каждая секция:**

- **`embedding`** — Как воспоминания превращаются в векторы для семантического поиска. `ollama` работает локально, `openai` вызывает облачные API. `nomic-embed-text` — хорошая локальная модель для Ollama.
- **`enrichment`** — Опциональный шаг LLM который улучшает воспоминания перед сохранением (лучшие summaries, авто-теги). Установите `none` чтобы пропустить.
- **`context`** — Управляет как воспоминания извлекаются при старте сессии. `auto` использует векторный поиск когда эмбеддинги доступны, откатывается на ключевые слова. `topup_recent` также включает недавние воспоминания чтобы у агента был свежий контекст.

## Использование

Ваша память работает автоматически. В начале сессии EchoVault извлекает релевантный контекст. В конце — сохраняется всё важное.

### Команды памяти

```bash
# Найти связанные воспоминания
memory search "authentication"

# Сохранить решение
memory save \
  --title "Switched to JWT auth" \
  --what "Replaced session cookies with JWT" \
  --why "Needed stateless auth for API" \
  --impact "All endpoints now require Bearer token" \
  --tags "auth,jwt" --category "decision"

# Показать контекст текущего проекта
memory context --project

# Получить полные детали по ID
memory details <id>
```

### Финализация сессии / Session Finalization

Перед завершением ЛЮБОЙ сессии с осмысленной работой вы ДОЛЖНЫ сохранить уроки. Это не опционально.

**Шаги для финализации:**

```bash
# 1. Обзор проделанного
memory sessions

# 2. Сохранить решения и уроки
memory save \
  --title "Итог сессии / Session summary" \
  --what "Что было достигнуто / What was accomplished" \
  --why "Обоснование и решения / Reasoning and decisions" \
  --impact "Внесённые изменения / Changes made" \
  --tags "summary,session" \
  --category "context"

# 3. Проверить что контекст сохранён
memory context --project
```

**Что сохранять в конце сессии:**

| Сохранить | Не сохранять |
|-----------|--------------|
| Архитектурные решения | Тривиальные изменения (опечатки) |
| Исправленные баги (с причиной) | Инфо очевидная из кода |
| Обнаруженные паттерны | Дубликаты (ищите сначала!) |
| Изменения инфраструктуры | |
| Полученные уточнения пользователя | |
| Следующие шаги / TODO | |

**Всегда используйте флаг `--source`** для идентификации агента: `claude-code`, `codex`, или `cursor`.

### Терминальный дашборд

```bash
memory dashboard
memory dashboard --project my-project
```

Используйте для:
- просмотра воспоминаний
- предпросмотра деталей
- редактирования в `$EDITOR`
- архивирования или восстановления
- просмотра дубликатов
- запуска импорта, переиндексации, обновления
- командной палитры

**Клавиши:**

| Клавиша | Действие |
|---------|----------|
| `↑↓` или `jk` | Навигация |
| `Enter` | Просмотр деталей |
| `e` | Редактировать в `$EDITOR` |
| `d` | Архивировать / восстановить |
| `D` | Дубликаты |
| `a` | Архивировать или восстановить |
| `m` | Слить дубликаты |
| `x` | Оставить дубликаты раздельно |
| `i` | Импорт из хранилища |
| `R` | Переиндексация векторов |
| `r` | Обновить данные |
| `:` | Командная палитра |
| `?` | Окно помощи |
| `q` | Выход |

## Как это работает

```
~/.memory/
├── vault/                    # Markdown для Obsidian
│   └── my-project/
│       └── 2026-02-01-session.md
├── index.db                  # SQLite: FTS5 + sqlite-vec
└── config.yaml               # Конфиг провайдера эмбеддингов
```

Каждое воспоминание — Markdown файл с YAML frontmatter. Obsidian может открыть `~/.memory/vault/` напрямую.

Секреты редактируются перед сохранением:
1. Явные `<redacted>` теги
2. Паттерны (API ключи, токены, пароли)
3. Кастомные `.memoryignore` правила

## Команды

```bash
memory save [options]         # Сохранить воспоминание
memory search <query>         # Поиск по всем проектам
memory context [--project]    # Загрузить релевантный контекст
memory details <id>           # Показать полные детали
memory dashboard              # Открыть TUI дашборд
memory setup <agent>          # Установить MCP конфиг
memory init                   # Создать структуру директорий
memory reindex                # Перестроить поисковый индекс
memory config init            # Сгенерировать стартовый конфиг
memory sessions               # Список файлов сессий
memory import <path>          # Импортировать внешние MD файлы
```

## Удаление

```bash
pip uninstall echovault
rm -rf ~/.memory
```

Чтобы удалить MCP конфиг агента:

```bash
memory uninstall claude-code   # или cursor, codex, opencode
```

## Приватность

- Всё хранится локально в `~/.memory/`
- Никаких внешних API вызовов по умолчанию
- Опциональные эмбеддинги через Ollama (локально) или OpenAI (облако)
- Секреты редактируются перед сохранением
- Никакого трекинга, аналитики или телеметрии
